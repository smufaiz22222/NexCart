# Sample Recommendation Evaluation Report

Use this report format for final-year documentation and viva demonstrations.

## Run Metadata

- Report type: offline benchmark
- K values: 5 and 10
- Data source: `Order`, `OrderItem`, `RecommendationInteraction`, `ProductSimilarity`
- Evaluation method: leave-one-out purchase test
- Generated by: `npm run recommendations:benchmark`

## Metrics Explained

| Metric | Meaning |
| --- | --- |
| Precision@K | Fraction of top-K recommendations that contain the hidden purchase. |
| Recall@K | Whether the hidden purchase appears in the top-K list. |
| MAP@K | Rewards placing the hidden purchase higher in the ranked list. |
| NDCG@K | Rank-sensitive relevance score. |
| Coverage | Fraction of catalog appearing in recommendations. |
| Diversity | Category spread among recommended products. |

## Interpretation

- Higher precision means fewer irrelevant recommendations.
- Higher recall means more hidden purchases are recovered.
- Higher MAP/NDCG means useful recommendations appear earlier.
- Higher coverage means the recommender is not only showing a tiny part of the catalog.
- Higher diversity means the system recommends across more categories.
